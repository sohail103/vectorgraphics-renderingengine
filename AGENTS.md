# AGENTS.md - Vector Graphics Rendering Engine

## Project Overview

ESP32-based vector graphics rendering engine using onboard I2S DACs to drive a CRT oscilloscope in XY mode. Outputs analog voltages to draw animations on the oscilloscope screen.

## Build Commands

```bash
pio run          # Build the project
pio run -t upload  # Build and flash to ESP32
pio run -t monitor # Build and monitor serial output
```

Alternatively with ESP-IDF:
```bash
idf.py build
idf.py flash monitor
```

## Architecture

- **Target**: ESP32 WROOM32 (board: upesy_wroom, platform: espressif32)
- **Framework**: ESP-IDF 5.5.0 via PlatformIO
- **Sample Rate**: 60,000 Hz
- **Resolution**: 8-bit per channel (0-255, centered at 127)
- **Max Points per Frame**: 8192

## Drawing API

All drawing functions live in `src/core/draw_utils.c`. The global `wave[]` buffer holds interleaved X/Y uint16 samples (8192 points max).

### Current Primitives
| Function | Purpose |
|----------|---------|
| `add_point(x, y)` | Add single point to frame buffer |
| `draw_line(x1, y1, x2, y2, steps)` | Line with interpolation |
| `draw_circle(cx, cy, r, points)` | Circle outline |
| `blank_for(n)` | Insert n blank points (beam off) |

### Primitives to Add (SVG Support)
| Function | Purpose |
|----------|---------|
| `draw_ellipse(cx, cy, rx, ry, points)` | Ellipse (SVG `<ellipse>`, `<circle>`) |
| `draw_arc(cx, cy, rx, ry, start_angle, end_angle, points)` | Elliptical arc |
| `draw_bezier_quadratic(x0, y0, cx, cy, x1, y1, steps)` | Quadratic bezier (Q command) |
| `draw_bezier_cubic(x0, y0, c1x, c1y, c2x, c2y, x1, y1, steps)` | Cubic bezier (C command) |
| `draw_rect(x, y, w, h, points)` | Rectangle (SVG `<rect>`) |
| `reset_frame()` | Clear wave buffer for new frame |

## Animation System

8 animations defined in `src/animations/`:

| Mode | File | Description |
|------|------|-------------|
| `MODE_CUBE` | cube.c | 3D rotating wireframe cube |
| `MODE_FACE` | face.c | Animated face with blinking eyes |
| `MODE_ORBIT` | orbit.c | Solar system with moons |
| `MODE_SPIRO` | spirograph.c | Spirograph/hypotrochoid curve |
| `MODE_TUNNEL` | tunnel.c | 3D tunnel with twist and waves |
| `MODE_DONUT` | donut.c | 3D torus (donut shape) |
| `MODE_LISSAJOUS` | lissajous.c | 3D Lissajous curve projection |
| `MODE_CAT` | cat.c | Animated cat silhouette |

To change the active animation, edit `ANIMATION_MODE` in `src/main.c:6`.

## SVG Support (Planned)

### Architecture: Host-side Compilation
SVG parsing happens on the host machine. Output is a compact array of drawing commands that gets compiled into the firmware. **No SVG parser runs on ESP32.**

### Flow
```
SVG File → Host Tool (Python) → C source with command array → Compile → Flash
```

### Tool: svg2c (Python script)
Parses SVG using NanoSVG, generates C code:
- Command enum array (`DRAW_LINE`, `DRAW_CIRCLE`, `DRAW_ELLIPSE`, `BEZIER_CUBIC`, etc.)
- Parameter struct array with coordinates/control points
- Auto-scales to 255x255 space
- Resolves viewBox and basic transforms

Example output:
```c
// Generated by svg2c from input.svg
typedef enum { CMD_LINE, CMD_CIRCLE, CMD_BEZIER_CUBIC } DrawCmd;
static const DrawCmd commands[] = { CMD_LINE, CMD_CIRCLE, ... };
static const float params[] = { 10, 20, 100, 50, ... };
```

### Library: NanoSVG (memononen/nanosvg)
- Single header (~3000 lines), zlib license
- Parses SVG to cubic bezier shapes
- Only the **parser** portion needed

### Supported SVG Elements
1. `<path>` - Full path (M, L, H, V, C, S, Q, T, A, Z)
2. `<circle>`, `<ellipse>` - Basic shapes
3. `<rect>` - Rectangles
4. `<line>`, `<polyline>`, `<polygon>` - Line primitives
5. `<g>` - Groups with transforms

### Supported Transforms
- `viewBox` coordinate mapping
- Basic transforms: `translate`, `scale`, `rotate`

### Out of Scope
- Gradients, patterns, masks, filters
- Text, images, CSS styling

## Implementation Order

1. **Add drawing primitives** - ellipse, arc, bezier curves, rect
2. **Add NanoSVG library** - Copy header to lib/nanosvg
3. **Create svg_renderer module** - Executes pre-compiled SVG command arrays
4. **Create svg2c.py tool** - Host-side SVG parser using NanoSVG
5. **Test the pipeline** - svg2c generates C → compile → flash → verify on scope
6. **Add transform support** - viewBox, translate, scale, rotate
7. **Memory optimization** - Handle SVGs with >8192 points via chunking
